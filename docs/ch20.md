> **本章一句话**:FDE 是"技术 + 业务 + 交付 + 软技能"四维复合能力——这一章给出可落地的能力模型、培养路径和晋升阶梯,让人力资源、转型者、培养者都有章可循。

## 20.1 FDE 能力金字塔

FDE 的能力,可以画成一座四层金字塔:

```
        /\
       /软 \   软技能:沟通/抗压/owner/学习
      /------\
     / 交付  \  交付与现场:MVD/灰度/信任/知识转移/项目管理
    /----------\
   /   业务    \ 业务与行业:行业认知/痛点洞察/合规/客户语言
  /--------------\
 /     技术      \ 技术:Python/LLM/RAG/Agent/数据工程/部署
/------------------\
```

底座是**技术**(必要条件),往上依次是**业务**、**交付**、**软技能**。越往上越稀缺、越难速成、也越值钱。一个合格的 FDE,四层都不能有明显短板;一个优秀的 FDE,在某一两层特别突出。

## 20.2 技术能力清单

技术是 FDE 的底座,2026 年的 FDE 技术栈(对应[第 9 章](/ch09)):

**必备(所有 FDE)**
- **Python**:FDE 的母语,数据处理、模型调用、脚本;
- **LLM 应用**:调用 LLM API、prompt 工程、上下文工程;
- **RAG**:向量库、嵌入、检索、重排;
- **Agent**:LangGraph/CrewAI、工具调用、MCP;
- **数据工程**:ETL、Pandas/Spark、SQL、数据质量;
- **部署基础**:Docker、Linux、bash、基本云。

**进阶(高级 FDE)**
- **推理服务**:vLLM、量化、性能优化;
- **MLOps**:模型管理、监控、AB;
- **多 Agent 编排**:复杂编排、状态管理、HITL 设计;
- **可观测**:LLM 链路追踪、评估;
- **安全**:prompt injection 防护、数据脱敏。

**行业相关(按行业)**
- 金融:评分卡、LightGBM、风控引擎、SHAP/PSI;
- 医疗:NER、HL7/FHIR、ICD 编码;
- 制造:YOLO、TensorRT、PLC/OPC UA;
- 物流:OR-Tools、强化学习、实时流;
- 能源:时序预测、SCADA、MCP server。

> **技术能力的现实**:FDE 不需要在每项技术上都达到专家级,但必须"**端到端能打通**"。一个只会调 API 但不会部署、不懂数据的 FDE,在客户现场会寸步难行。

## 20.3 业务与行业能力

业务能力是 FDE 区别于纯工程师的关键。

**核心要素**
- **行业认知**:该行业的格局、价值链、关键指标、监管、典型流程;
- **痛点洞察**:能从混沌业务里识别真痛点(Discovery-first);
- **客户语言**:用客户的业务语言沟通,而非技术黑话;
- **合规意识**:知道该行业的合规红线;
- **价值量化**:能把技术方案翻译成业务价值(ROI)。

**如何积累行业能力**
- **浸泡**:在客户现场/行业里待足够久;
- **访谈**:和行业从业者深度交流;
- **阅读**:行业报告、监管文件、标杆案例;
- **复盘**:每个项目后沉淀行业认知。

> **行业能力的稀缺性**:技术可以速成(学几个月 Python/RAG),但行业能力需要年复一年的积累。这也是为什么"懂某个行业的 FDE"比"纯技术 FDE"更值钱、更难被替代。

## 20.4 交付与现场能力

交付能力是 FDE 把"能力"变成"结果"的桥梁。

**核心要素**
- **MVD 思维**:快速跑通最小系统;
- **灰度能力**:设计安全的放量策略;
- **项目管理**:周计划、阶段门、风险管理;
- **客户管理**:预期管理、需求边界、关系维护;
- **知识转移**:文档、培训、共建、陪跑;
- **现场生存**:融入、谦逊、靠谱([第 10 章](/ch10))。

## 20.5 软技能:FDE 的隐形分水岭

软技能往往是 FDE 能走多远的决定因素,也是最容易被忽视的。

**核心软技能**
- **沟通**:对客户(业务语言)、对内部(技术语言)、对高层(价值语言)三套话术;
- **抗压**:现场高压、客户冲突、延期,保持冷静;
- **owner 心态**:对结果负责到底,不甩锅;
- **学习力**:快速学习新行业、新技术;
- **同理心**:理解客户、一线、团队的立场;
- **判断力**:在不确定环境里做决策。

> **软技能的真相**:很多技术很强的工程师转型 FDE 失败,不是技术不够,而是软技能(沟通、抗压、owner)不足。FDE 的工作高度依赖"和人打交道",软技能是硬门槛。

## 20.6 培养路径:三种主流来源 + 培养计划

**三种主流来源的转型路径**

1. **工程师 → FDE(最常见)**:补业务 + 交付 + 软技能。建议:先在 FDE 团队做"技术 leg",跟着资深 FDE 做 2—3 个项目,逐步承担 Context/Feedback;
2. **咨询/SA → FDE**:补技术(动手能力)。建议:强化 Python + LLM/RAG/Agent 实操,在项目里从"出方案"过渡到"出系统";
3. **行业专家 → FDE(最稀缺)**:补技术基础。建议:学 Python + AI 应用,做"行业 + AI"复合 FDE。

**培养计划设计(参考"锋刃计划"式)**
- **第一阶段(基础,4—8 周)**:AI 技术底座(LLM/RAG/Agent/部署)+ FDE 方法论(CDEF);
- **第二阶段(沙盘,4—8 周)**:模拟项目,跑通 CDEF 全流程;
- **第三阶段(影子,8—12 周)**:跟随资深 FDE 做真实项目,观察学习;
- **第四阶段(半独立,8—12 周)**:在指导下承担 CDEF 部分阶段主责;
- **第五阶段(独立)**:独立扛小单,完整交付。

**认证与评估**
- 阶段门考核(每个培养阶段有过关标准);
- 真实项目交付评估;
- 行业 FDE 认证(部分公司/机构推出)。

## 20.7 能力评估与晋升阶梯

**FDE 晋升阶梯(对标[第 2 章](/ch02)薪酬)**
- **Junior FDE**:在指导下完成 Engineer 阶段任务;
- **Mid FDE**:独立完成 Engineer + 部分 Design;
- **Senior FDE**:主导中型项目全 CDEF,带 1—2 个 junior;
- **Staff FDE**:主导大型项目,影响方法论/平台,带团队;
- **Principal/Lead**:战略级,影响公司 FDE 体系。

**评估维度**
- **交付结果**:项目 ROI、客户满意度;
- **能力广度/深度**:技术 + 业务 + 交付 + 软技能;
- **影响力**:对团队、方法论、平台的贡献;
- **行业纵深**:在特定行业的不可替代性。

## 本章小结

- FDE 能力金字塔四层:技术(底座)→ 业务 → 交付 → 软技能(顶),越往上越稀缺越值钱;
- 技术:Python/LLM/RAG/Agent/数据工程/部署必备,推理/MLOps/编排/可观测/安全进阶,行业技术按需;
- 业务:行业认知/痛点洞察/客户语言/合规/价值量化,靠浸泡+访谈+复盘积累;
- 交付:MVD/灰度/项目管理/客户管理/知识转移/现场生存;
- 软技能:沟通/抗压/owner/学习/同理心/判断力,是隐形分水岭;
- 培养路径:工程师/咨询/行业专家三种来源,基础→沙盘→影子→半独立→独立五阶段;
- 晋升:Junior→Mid→Senior→Staff→Principal,按交付结果+能力+影响力+行业纵深评估。
